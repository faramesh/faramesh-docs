## What is Faramesh?

Faramesh sits between your AI agent and the tools it can call. Every tool invocation passes through a policy engine that returns one of three answers:

<CardGrid>
  <Card title="Permit" icon="approve-check">
    The rule said yes. The action runs. A signed decision record lands in the audit ledger.
  </Card>
  <Card title="Deny" icon="error">
    Blocked before execution. The agent receives a structured denial telling it why and what to do next.
  </Card>
  <Card title="Defer" icon="warning">
    Held for a human to approve in the inbox. Nothing runs until the operator decides.
  </Card>
</CardGrid>

There is no language model on the enforcement path. Policy is compiled to a deterministic AST and evaluated by the daemon. The agent never sees raw credentials — providers mint short-lived, scoped tokens at the moment of execution.

## Get going

<Steps>

1. **Install the CLI.**

   <Tabs>
     <TabItem label="curl">
       ```bash
       curl -fsSL https://raw.githubusercontent.com/faramesh/faramesh-core/main/install.sh | bash
       ```
     </TabItem>
     <TabItem label="npm">
       ```bash
       npx @faramesh/cli@latest
       ```
     </TabItem>
     <TabItem label="Homebrew">
       ```bash
       brew install faramesh/tap/faramesh
       ```
     </TabItem>
   </Tabs>

2. **Generate `governance.fms`** in your agent project.

   ```bash
   faramesh init
   ```

3. **Compile and start enforcement.**

   ```bash
   faramesh check
   faramesh apply
   ```

4. **Wire your agent.** Drop in the SDK shim or point an MCP client at the proxy port.

   See the [frameworks index](/frameworks/) for a one-page per framework.

</Steps>

## A complete policy in one page

Below is a real `governance.fms`. Faramesh compiles it once at `faramesh apply` and the daemon enforces every block live.

```fpl
import "registry.faramesh.dev/frameworks/langgraph@1.0.0"
import "registry.faramesh.dev/policies/pci-dss@1.2.0" as pci

runtime {
  mode    = "enforce"
  wal_dir = "./faramesh-wal"
  backend = "sqlite"
}

provider "vault" {
  type      = "vault"
  addr      = env("VAULT_ADDR")
  token     = env("VAULT_TOKEN")
  mount     = "kv/data/payments"
}

agent "payments-bot" {
  default deny

  rules {
    permit stripe/charge   if amount < $500
    defer  stripe/refund
    deny   stripe/payouts
    permit health/*
  }

  rate_limit "stripe/*": 10 per minute

  redact stripe/charge args: ["card_number", "cvv"]

  budget daily {
    max       $500
    warn_at   0.8
    on_exceed deny
  }

  egress {
    allow = ["api.stripe.com", "api.faramesh.dev"]
  }

  alert {
    on     = "deny"
    notify = "slack://#sec-alerts"
  }
}
```

When one stack is no longer enough, **Faramesh Cloud** is the hosted operating console for the same runtime — orgs, projects, an approvals UI, an evidence browser, and fleet visibility.

The daemon stays the enforcement truth. Cloud is the surface security and platform teams operate through.
